# 計算の高速化
v22.1 DMRG 励起状態 -> 重電荷窓　の計算が31時間経っても終わりません。これはpythonで計算しているからですか？
今後は計算はrustで行い、その結果をpythonでまとめたり可視化したりするようにしてください。pythonには重い処理はさせないようにしましょう。rustのプログラムはcpuの数だけマルチスレッドで高速化させたい。ただし、並列処理によって結果がかわらないようにすること。冪等性を守ることがマルチスレッド化で難しくなるのであれば、無理にマルチスレッド化しなくて良い。
現在のpythonプログラムをこのまま続けるよりも、rustで計算を行った方が速いのであればプログラムを作り直して最初からやり直しましょう。

# 独立して行える研究の同時実行
この計算結果を待たずに進められる研究はありますか？
複数の研究プログラムをあらかじめ作り、それぞれはシングルスレッド動作で、複数の研究をマルチプロセスで実行することも考えてください。
