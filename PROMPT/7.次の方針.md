## 1. v25.2の解析的中心部分は、既に解けています

v25.1では、3次元staggered fermionの各横運動量ブロックを、質量 (\pm\mu) の1次元staggered chainに厳密還元し、その半無限領域のmodular Hamiltonian係数を

[
K_{\mathrm{NN}}(\xi)\simeq \pi g(\mu),\xi,
]

[
K_{\mathrm{diag}}(\xi)\simeq
2\pi\mu,g_m(\mu)\left(\xi-\frac12\right)
]

として数値抽出しています。リポジトリでは

[
g(0.5)\simeq0.9450,\quad
g(1)\simeq0.8346,\quad
g(\sqrt2)\simeq0.7457
]

が得られています。([GitHub][2])

一方、Eislerのstaggered-potential fermion chainの厳密解では、半無限鎖のentanglement Hamiltonianは

[
H_E=4\kappa,\mathbf K(\kappa'),T,
]

[
\kappa=\frac{1}{\sqrt{1+\mu^2}},
\qquad
\kappa'=\frac{|\mu|}{\sqrt{1+\mu^2}},
]

となります。ここで (\mathbf K) は第一種完全楕円積分で、(T) の最近接成分は (-m/2)、staggered diagonal成分は ((-1)^m\mu(m-\tfrac12)) です。符号の差は (c_n\mapsto(-1)^n c_n) という局所ユニタリ変換で吸収できます。([arXiv][3])

したがって、リポジトリの規格化に写すと厳密に

[
\boxed{
g(\mu)=g_m(\mu)
=\frac{2\kappa,\mathbf K(\kappa')}{\pi}
}
]

です。

さらにGaussのAGM公式から、

[
\boxed{
g(\mu)
======

\frac{1}{\operatorname{AGM}
\left(1,\sqrt{1+\mu^2}\right)}
}
]

とも書けます。積分表示は

[
\boxed{
g(\mu)
======

\frac{2}{\pi}
\int_0^{\pi/2}
\frac{d\theta}
{\sqrt{1+\mu^2\cos^2\theta}}
}
]

です。

これを評価すると、

|    (\mu) |     厳密式からの (g(\mu)) |
| -------: | ------------------: |
|    (0.5) | (0.945006330929758) |
|      (1) | (0.834626841674073) |
| (\sqrt2) | (0.745749187316330) |

となり、v25.1の数値と一致します。

したがって、**1次元関数 (g(\mu)) の発見そのものは新規性として主張すべきではありません**。先行研究の厳密解を3次元staggered reductionに接続したこと、ならびにそこから3次元の異方的BZ momentを導くことが、新規性候補です。先行研究自身も、massive lattice fermionにおけるBW係数が分散関係に依存する非普遍的prefactorになることを示しています。([arXiv][3])

## 2. v25.1で想定されていた漸近形も確定できます

厳密式を展開すると、小質量では

[
\boxed{
g(\mu)
======

1-\frac{\mu^2}{4}
+\frac{9\mu^4}{64}
-\frac{25\mu^6}{256}
+\frac{1225\mu^8}{16384}
+O(\mu^{10})
}
]

です。

したがって、v25.2候補として挙げられていた

[
\mu^2\log\mu
]

型の非解析項はありません。少なくとも熱力学的半無限鎖の (g(\mu)) については、原点近傍で (\mu^2) の解析的級数になります。

大質量では、

[
\boxed{
g(\mu)
\sim
\frac{2\log(4|\mu|)}{\pi|\mu|}
}
]

です。

従って、単純な (1/\mu^2) ではなく、また単純な (1/\mu) でもなく、

[
\frac{\log|\mu|}{|\mu|}
]

が先頭です。v25.1文書で挙げられている数値的な小質量・大質量候補は、厳密式によって置き換えるべきです。([GitHub][2])

さらに、分散

[
E(q)=\sqrt{\cos^2q+\mu^2}
]

の複素零点から推定される相関長スケールは

[
\xi_{\mathrm{corr}}\sim\frac{1}{\operatorname{arsinh}|\mu|}
]

です。したがって (\mu=10^{-3}) では (\xi_{\mathrm{corr}}\sim10^3) となり、(N=256) の有限鎖では熱力学極限の小質量漸近を直接判定できません。この非一様極限が、数値から誤った非解析形を推測しやすい原因です。これは現在の有限鎖計算を否定するものではなく、**有限鎖計算を厳密式への収束試験として再定義すべき**という意味です。

## 3. 3次元BZ平均も、現在の値まで解析的に閉じられます

リポジトリの還元では

[
\mu(k_y,k_z)
============

\sqrt{\cos^2 k_y+\cos^2 k_z}
]

です。厳密な (g(\mu)) を使うと、

[
r_x
===

\left\langle
g!\left(
\sqrt{\cos^2 k_y+\cos^2 k_z}
\right)
\right\rangle_{\mathrm{BZ}},
]

[
r_\perp
=======

\left\langle
2\cos^2k_y,
g!\left(
\sqrt{\cos^2 k_y+\cos^2 k_z}
\right)
\right\rangle_{\mathrm{BZ}}
]

となり、

[
\lambda_x=\frac1{r_x},
\qquad
\lambda_\perp=\frac1{r_\perp}.
]

高精度評価すると、

[
r_x
===

0.843549215293854\ldots,
]

[
r_\perp
=======

0.813385882130203\ldots,
]

したがって

[
\boxed{
\lambda_x
=========

1.18546728734926\ldots
}
]

[
\boxed{
\lambda_\perp
=============

1.22942876434131\ldots
}
]

です。これはv25.1の直接計算と一致します。([GitHub][2])

### 異方性は数値事実ではなく定理にできます

[
Y=\cos^2k_y,\qquad Z=\cos^2k_z,
]

[
h(s)=g(\sqrt{s})
]

と置きます。(g) は

[
g'(\mu)
=======

-\frac{2\mu}{\pi}
\int_0^{\pi/2}
\frac{\cos^2\theta,d\theta}
{(1+\mu^2\cos^2\theta)^{3/2}}
<0
]

なので、(\mu>0) で厳密に減少します。従って (h(Y+Z)) は (Y) に関して減少します。

また (\mathbb E[Y]=1/2) なので、

[
r_\perp-r_x
===========

2\operatorname{Cov}
\left(
Y,h(Y+Z)
\right)
<0.
]

よって、

[
\boxed{
r_\perp<r_x
\quad\Longrightarrow\quad
\lambda_\perp>\lambda_x
}
]

です。

これは非常に良い論文上の中心命題です。単に「数値的に方向依存だった」ではなく、

> transverse current weightingとmass-dependent BW prefactorの負の共分散により、異方性の向きが必然的に決まる

と説明できます。

Rust側ではAGM実装が最も簡単です。

```rust
#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Mass(pub f64);

#[derive(Debug, Clone, Copy, PartialEq)]
pub struct BwPrefactor(pub f64);

pub fn arithmetic_geometric_mean(
    mut a: f64,
    mut b: f64,
) -> Result<f64, ExactGError> {
    if !a.is_finite() || !b.is_finite() || a <= 0.0 || b <= 0.0 {
        return Err(ExactGError::InvalidAgmArguments);
    }

    for _ in 0..128 {
        let next_a = 0.5 * (a + b);
        let next_b = (a * b).sqrt();

        if (next_a - next_b).abs()
            <= 8.0 * f64::EPSILON * next_a.abs().max(1.0)
        {
            return Ok(next_a);
        }

        a = next_a;
        b = next_b;
    }

    Err(ExactGError::NoConvergence)
}

pub fn exact_g(mass: Mass) -> Result<BwPrefactor, ExactGError> {
    let mu = mass.0;

    if !mu.is_finite() {
        return Err(ExactGError::NonFiniteMass);
    }

    let scale = (1.0 + mu * mu).sqrt();
    let agm = arithmetic_geometric_mean(1.0, scale)?;

    Ok(BwPrefactor(1.0 / agm))
}
```

最低限の性質テストは、

[
g(-\mu)=g(\mu),\quad
g(0)=1,\quad
0<g(\mu)\leq1,
]

および (\mu>0) での単調減少です。

### 現在の最大の弱点

一つのmicroscopic theoryからすべてが出ているのではなく、複数のtoy model、数値実験、既知理論の再現、候補的接続が同じアトラスに収容されています。

従って現状のQRNは、

> 一つの理論

というより、

> 一つの研究方法論と、関連候補モデルの検証基盤

です。

これはREADMEの自己位置づけとも整合します。([GitHub][1])

統一理論へ進むために必要なのは、モジュールをさらに増やすことではありません。次の仕様を満たす単一の `QRN-Core v1` が必要です。

* 一つの作用またはHamiltonian
* 有限個のmicroscopic parameters
* 明示されたHilbert space
* localityまたはgraph-locality
* gauge constraints
* continuum scaling parameter
* phase diagram
* 同じ極限からmatterとgeometryが同時に出る
* matterがgeometryにbackreactする
* 複数observableを同じparametersで予測する

「同じネットワーク表現で書ける」「同じ圏論的語彙で説明できる」だけでは統一ではありません。統一を示すのは、**同じmicroscopic dynamics、同じRG flow、同じ固定点、同じパラメータによるcross-domain constraints**です。

### PASS数の扱い

852 PASS / 0 FAILのような総数はsoftware healthの指標にはなりますが、物理理論の証拠量ではありません。([GitHub][1])

今後はPASSを少なくとも次に分けるべきです。

* implementation regression
* mathematical identity
* internal consistency
* known-physics benchmark
* numerical convergence
* negative control
* frozen novel prediction
* independent external reproduction

総数を表示する場合も、最後の二つを別枠にしてください。既知結果を100個再現することと、新しい凍結予測を1個当てることは、認識論的には同じ1 PASSではありません。

## 9. AI実装を前提とした検証設計

LLMに実装とcheckerの両方を書かせると、同じ概念的誤解が両方へ複製される可能性があります。言語やモデルを変えるだけでは、訓練データ由来の相関が残ります。

そのため、今後の重要結果には次の四層を要求するべきです。

1. **仕様層**
   数式、規格化、入力領域、失敗条件をコードより先に凍結する。

2. **構成的実装層**
   数式を直接実装する。今回ならAGM式。

3. **独立特徴づけ層**
   別の数学的特徴づけを実装する。今回なら直接積分または有限鎖相関行列。

4. **破壊層**
   mutation testing、符号反転、規格化変更、BZ measure変更などに対し、必ずテストが失敗することを確認する。

特に「二つのコードが一致した」だけでなく、

> どの誤りを入れると、どのfalsifierが反応するか

をcertificateに入れると、QRNの研究方法論自体が一つの成果になります。

## 推奨する実行順序

1. **既知厳密解を取り込み、(g=g_m) を確定する**
2. **BZ積分を区間認証し、異方性を定理化する**
3. **有限サイズgateと判定分岐を修正する**
4. **v25.2-finalを凍結する**
5. **anomaly-searchのタイトルと定理のscopeを修正し、最初の外部査読単位にする**
6. **modular-Hamiltonian結果を独立した短い論文にする**
7. **flavor三部作をadaptive model developmentとして統合する**
8. **その後にのみ、(\Gamma[g]) とstress tensorから誘導重力を検討する**
9. **新しい広範なQRNモジュールの追加は、`QRN-Core v1` が定義されるまで停止する**

最も価値の高い次の一手は、壮大な新章を追加することではありません。**先行研究によって解析的に解けた (g(\mu)) を正しく取り込み、3次元BZ momentと異方性定理を新規成果として切り分け、v25系列を閉じて査読可能な固定物にすることです。**
