## まず現在地の評価

この研究の強みは、通常の「思弁的統一理論」と違い、再現可能な Rust 実験、Lean 定理、主張依存 DAG、反証条件、予言台帳を持っている点です。最新版では v17.2 まで進み、CP 破れ・複素構造・測度問題に焦点が移っています。v17.0 は未解決残高として、測度問題、自由度増加の起源、3+1D の QNEC→非線形 Einstein、MPS/疎対角化、SU(2)/SU(3)、形式化、出版、BMV などの外部実験を挙げています。([GitHub][2])

したがって、次の研究フェーズの目的は明確です。

**QRN が「多くの既存理論を再現する計算アトラス」から、「単一の力学原理から GR + 標準模型 + パラメータ分布 + 反証可能予言を出す理論」へ移ること**です。

そのための優先順位は次です。

## 1. 最優先は「測度問題」を主戦場にする

v17.0 で新たに開いた中心問題は、幾何・Wilson 線・世代対・複素構造 τ などの離散データの上に、どの測度を置くべきかという問題です。v17 系では、浅い質量比が過剰代表されており、それを補正する data-blind な測度補正が +3 nats 級で生き残る、という形まで来ています。これは単なるフィット改善ではなく、「なぜその幾何が選ばれるのか」という上位原理への入口です。([GitHub][2])

ここでやるべき具体作業は次です。

まず、模型空間 Ω を明示的に定義します。要素は、磁束、Wilson 線、世代 pairing、トーラス数、複素構造 τ、Higgs 幅 σ、境界条件、ゲージ群表現などです。その上で、候補測度を事前登録します。候補は少なくとも、単純一様測度、Jeffreys/Fisher 測度、MDL 測度、熱力学的測度、深さ補正 `exp(βD)`、ヒストグラム平坦化測度を含めるべきです。

次に、測度補正が本当に「データを見ずに」定義できるかを検査します。bin 幅やグリッド選択に依存するなら、それは原理ではなく後付けです。v17.1 の CP 論文草稿自身も、σ 連続周辺化、τ_im の未探索、config-density の解析形未解明を限界として挙げています。([GitHub][3])

ここでの理想的な v18 系は、次のようになります。

```text
v18.0  測度候補の事前登録
v18.1  σ_H の連続周辺化
v18.2  config-depth density の解析形推定
v18.3  τ_im / aspect ratio 走査
v18.4  測度候補の生存・棄却・未決を nats で判定
v18.5  測度問題の短報原稿
```

成功条件は、「測度補正が evidence を上げる」では足りません。**測度が観測値を使わずに構成され、グリッドを変えても順位を保ち、新しい holdout にも効く**ことが必要です。

## 2. 3+1D の Einstein 方程式への橋を作る

現在の重力側で最も危険な飛躍は、2D/AdS₃/BTZ 的な成功から 3+1D の重力へ外挿している部分です。v15.6 では一次・二次・BTZ 全次数の梯子が報告されていますが、v17.0 自身が「3+1D の QNEC → 非線形 Einstein」は未解決として残しています。([GitHub][1])

ここは統一場理論にとって避けられません。やるべきことは、単なる数値再現ではなく、次の形の橋渡しです。

```text
QRN state / relational entropy
        ↓
local modular Hamiltonian / relative entropy
        ↓
QNEC / energy condition
        ↓
effective stress tensor Tμν
        ↓
Einstein equation + higher curvature corrections
        ↓
3+1D Newtonian limit, gravitational waves, cosmology
```

特に必要なのは、**有効作用**です。

```text
S_eff[g, fields] =
∫ d⁴x √-g [
  (1/16πG) (R - 2Λ)
  + α R² + β R_{μν}R^{μν}
  + matter terms
  + gauge terms
  + cutoff-suppressed corrections
]
```

QRN からこれを読み出し、`G`, `Λ`, `α`, `β` がどのように決まるかを示す必要があります。もし高次曲率項が観測的に大きすぎるなら、C5 の中心テーゼは降格されます。

次の研究では、3+1D で少なくとも以下を検査対象にすべきです。

* Newton ポテンシャル `1/r` の創発
* 線形化重力波の 2 偏極
* ローレンツ不変性の主要次数での保護
* Bianchi 恒等式 / Ward 恒等式
* `∇μ T^{μν}=0`
* QNEC の連続極限
* `R²` 補正の上界

ここでの反証条件は明確にできます。連続極限で残差 `B ≠ 0` が残る、QNEC の負ギャップが縮まない、Newton 極限が出ない、ローレンツ違反が一次で残る、などです。リポジトリにはすでに FAL-QNEC、FAL-CONTINUUM、FAL-LORENTZ の文化があるので、これを 3+1D 重力に拡張するのが自然です。([GitHub][4])

## 3. Z₂ core から SU(3)×SU(2)×U(1) core へ進む

現在の QrnCoreV2 は、Z₂ 格子ゲージ + staggered fermion の Gauss 拘束を厳密に解き、閉じ込め、弦張力、弦切れ、幾何読み出しなどを同一状態から読んでいます。これは良い中間段階ですが、統一場理論を名乗るには **標準模型のゲージ群そのものを QRN core に載せる**必要があります。README でも v15.3 は Z₂ core として記録され、v17.0 は SU(2)/SU(3) を残課題にしています。([GitHub][1])

進め方は段階的でよいです。

```text
Z₂ gauge + fermion
  → compact U(1)
  → Schwinger model / anomaly test
  → SU(2) small lattice
  → SU(3) color toy
  → SU(3)×SU(2)×U(1) constrained Hilbert basis
  → chiral matter / anomaly cancellation / Yukawa sectorとの接続
```

ここで最重要なのは、**アノマリー探索で得た「SM が有限窓内で一意」という結果を、実際の dynamical core に接続すること**です。現状の anomaly-search 論文草稿は、有限窓内で SM 1 世代が最小・一意であり、三重実装・SHA-256 証明書・Lean 検証を持つ、と主張しています。これは強い C2 資産ですが、まだ「そのスペクトルが QRN dynamics から出る」こととは別です。([GitHub][5])

したがって次の目標は、

```text
有限探索で SM が一意
        +
QRN core の Gauss constraint / locality / entropy constraint
        ↓
SM gauge sector が実際に low-energy spectrum として出る
```

という橋です。

この橋ができると、QRN は「標準模型を後から載せた」理論ではなく、「標準模型以外が落ちる」理論に近づきます。

## 4. 湯川・CP セクターは「factor 5」から「予測帯」へ進める

特にニュートリノは重要です。リポジトリは第 2 の U(1) が B−L + ν_R に向かう、Majorana neutrino を予言する、という構造を持っています。これを PMNS と 0νββ の数値帯に落とせれば、QRN は外部実験に接続します。([GitHub][6])

## 5. BMV は「C>0」ではなく「(C,V) 平面とスケーリング」で勝負する

重力の量子性を検査する BMV 型実験は、QRN の外部反証として重要です。ただし現在の文献状況では、単に重力誘起 entanglement が見えたから即「量子重力」とは言いにくくなっています。Marletto–Vedral 型の元の議論は、重力場が entanglement を媒介できれば媒介系は quantum である、という情報理論的主張でした。([arXiv][7])

一方で 2025 年の Nature 論文は、QFT 的な物質記述を含めると classical gravity でも entanglement を生むモデルがあり得て、実験パラメータとスケーリングで区別すべきだと論じています。([Nature][8]) 2026 年の別解析では、Newton–Schrödinger 型など加法分離ポテンシャルを持つ半古典モデルは entanglement を作らず、BMV はモデル間の witness 振る舞いを区別できるとしています。([arXiv][9])

QRN の予言台帳が `C²+V²=1` の円周、古典包絡、位相スケーリングまで登録しているのは正しい方向です。今後は BMV を「量子重力の一発判定」として使うのではなく、**QRN / classical channel / semiclassical model / decoherence model を同じ (C,V,Δφ,m,d,Δx,τ) 空間で比較する判別問題**として扱うべきです。([GitHub][6])

## 6. 形式化は「列挙結果」から「係数の導出」へ進める

現状の Lean の使い方は、有限探索の完全性検査にかなり向いています。anomaly-search 草稿では、SM の一意性を三重実装、証明書、Lean native_decide で支えています。([GitHub][5])

次に形式化すべきなのは、探索の上流です。

```text
group representation
  → weight system
  → Dynkin index
  → anomaly coefficient
  → anomaly equation
  → finite search domain
  → theorem
```

今の探索が「与えた anomaly 方程式の解を全列挙した」なら、次は「その anomaly 方程式自体が群論から導かれる」ことを Lean で閉じるべきです。これは統一場理論にとって大きいです。なぜなら、物理的仮定と計算的列挙の境界がさらに上流へ押し上げられるからです。

特に形式化すべき対象は次です。

* SU(3), SU(2), U(1) の anomaly coefficient
* Witten SU(2) anomaly の有限条件
* conjugation / charge rescaling / permutation の同値類
* `J=0` structural zero の代数的証明
* index 3 が単一タワー制約を生む数論的 no-go
* `B−L` charge plane の Plücker 分類

これにより、C2 主張の信頼基盤が「Rust が正しく走った」から「定理として閉じている」へ移ります。

## 7. 計算基盤は MPS / 疎対角化 / 証明付き数値へ移す

v17.0 は N=72 が dense diagonalization では重すぎること、MPS/疎対角化が必要なことを明示しています。([GitHub][2]) 再現性ガイドは、現在の方針として外部依存なし、固定 seed、自己実装数値ライブラリ、self-test、PASS/FAIL を重視しています。([GitHub][10])

この方針は維持すべきですが、研究速度と検証性のために二層構造にするのがよいです。

```text
探索層:
  外部ライブラリ可。MPS, DMRG, sparse eigensolver, interval arithmetic, Python/Julia も可。
  目的は仮説発見。

監査層:
  Rust std only / deterministic / self-test / fixed seed / JSON artifacts。
  目的は最終数値の再現と主張台帳への登録。
```

MPS を入れる場合も、最初から巨大な一般実装にしない方がよいです。まずは次の最小セットで十分です。

```text
MPS canonical form
two-site SVD update
TEBD imaginary time
DMRG ground state
entanglement entropy readout
gauge constraint projector
small-system exact diagonalizationとの一致テスト
```

すべての MPS 結果には、同じサイズで exact diagonalization と比較する regression anchor を付けるべきです。リポジトリの貢献規約にある「全シミュレーションに PASS/FAIL、陰性対照、主張台帳、失敗の記録」を MPS にもそのまま適用します。([GitHub][11])

## 8. 出版は「統一場理論」ではなく、切り出した強い結果から行う

すぐに「統一場理論」として出すより、まず次の 3 本を独立論文として出すべきです。

1 本目は anomaly-search。これは有限窓内の SM 一意性、三重実装、証明書、Lean 検証という、外部査読に耐えやすい内容です。([GitHub][5])

2 本目は geometric-yukawa。ランダムな O(1) 係数なしに、磁化トーラスの zero-mode overlap から質量階層と CKM を出す、という計算物理として出せます。現行草稿も JHEP / PRD を対象誌として想定しています。([GitHub][12])

3 本目は CP-complex-structure。長方形模型の structural zero、tilted flux の失敗、complex structure による CP 生成、+306 nats の反転、測度問題への移行という物語が完成しています。v17.2 は書誌照合まで進み、残りを LaTeX・図版・独立照合としています。([GitHub][3])

ここで重要なのは、各論文で「QRN が統一場理論である」と言い切らないことです。代わりに、

```text
This is a falsifiable computational mechanism within a larger QRN program.
```

という形にした方が通りやすいです。統一場理論の主張は、GR 側、SM gauge core 側、測度原理側が閉じてからでよいです。

## 9. 統一場理論を名乗るための最小条件

QRN が「統一場理論に到達した」と言うには、最低でも次の 5 条件が必要です。

第一に、単一の基礎自由度と単一の力学から、時空・重力・ゲージ場・物質・世代・湯川が別々の toy model ではなく同時に読み出されること。

第二に、連続極限で `GR + SU(3)×SU(2)×U(1) + chiral matter + Yukawa + neutrino sector` が出ること。

第三に、標準模型の自由パラメータが「後から選んだ離散データ」ではなく、データ非参照の測度または選択原理から分布として出ること。

第四に、少なくとも数個の非自明な holdout 予言が事前登録され、外部データで採点可能であること。

第五に、重要な C5 解釈が C4/C3/C2 に降格、つまり「解釈」から「現象論」「機構」「制限付き定理」へ押し下げられていること。

現状の QRN はこの方向にかなりよく進んでいますが、まだ最大の穴は、**測度原理、3+1D 重力、標準模型 gauge core、自由度増加の起源**です。v17.0 の残高表と一致します。([GitHub][2])

## 進行順序の提案

実務的には、次の順番が最も効率的です。

```text
v18: 測度問題 + σ連続周辺化 + τ_im + CP論文完成
v19: MPS/疎対角化基盤 + N=72 + lepton/PMNS holdout
v20: 3+1D QNEC→Einstein bridge + 有効作用読み出し
v21: U(1)→SU(2)→SU(3) constrained core
v22: SM spectrum + gravity readout を同一 core で実演
v23: 予言台帳の外部実験/観測との採点
```

この順番の理由は単純です。測度問題を解かないと、離散データが「選ばれた」のか「選んだ」のか分かりません。3+1D 重力を解かないと、統一場理論ではなく量子情報的重力 toy model に留まります。SU(3)×SU(2)×U(1) core を作らないと、標準模型の一意性探索と QRN dynamics が接続しません。

最も重要な一文に圧縮すると、次です。

**QRN の次段階は、成功例を増やす段階ではなく、依存 DAG の最深の仮定を一つずつ殺しに行き、生き残ったものだけを原理に昇格させる段階です。**
