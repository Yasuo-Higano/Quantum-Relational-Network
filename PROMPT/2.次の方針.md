---

**次は「シミュレーションを増やす」段階ではなく、「統一理論として弱い箇所だけを狙って潰す」段階**です。QRN はすでに、既存物理の再現・toy model・有限探索・Lean 形式化・論文化準備まで進んでいます。したがって今後は、次の 5 本柱に絞るのが最短です。

1. **単一 core を相互作用・ゲージ・物質まで拡張する**
2. **有限探索の主張を Lean/Coq でさらに定理化する**
3. **「自由に選んだ離散データ」を上位原理から選ぶ**
4. **QNEC/相対エントロピーから非線形 Einstein 方程式へ進む**
5. **QRN 固有の数値予言を作る**

現在の QRN は、README では「完成した万物理論ではなく、量子情報・創発時空・ゲージ制約・標準模型的スペクトルの再現可能な executable atlas」と自己位置づけし、主張を C0〜C5 に分類している点が非常に良いです。これは、物理理論として最も危険な「再現・機構・解釈・予言の混同」を避ける設計になっています。([GitHub][1])

一方で、v15.0 の時点でも未解決残高は明確に残っています。特に「自由度増加の起源」「σ・磁束・Wilson 線・トーラス数を一括選択する上位原理」「QNEC から非線形 Einstein 方程式」「重力以外の QRN 固有予言」「格子指数の数論」が研究の最前線として挙げられています。([GitHub][2]) したがって、今後の研究はこのリストをそのまま優先順位表にするのがよいです。

## まず現在地の評価

強い点は 4 つあります。

第一に、QRN は既存物理の「説明文」ではなく、Rust シミュレーション、結果ファイル、主張台帳、Lean 証明を持つ計算研究になっています。v15.0 では 59 本の Rust シミュレーション、Rust 314 PASS / 0 FAIL、Lean 検証が記録されています。([GitHub][2])

第二に、有限探索の主張、特に標準模型 1 世代の最小性・一意性・孤立性については、Rust の全数探索から Lean 定理化へ進んでいます。v6.8 では、SM が解であること、超電荷をずらすと破れること、指定された有限領域で SM 軌道だけが解であることを Lean で扱っています。([GitHub][3]) v14.5 では、大表現を含む E1 領域で成分数分布 `{15:1, 16:8, 17:1, 18:18, 22:2, 24:459}` を Lean 定理として検証し、SM がその窓で最小・一意だとしています。([GitHub][4])

第三に、単一 core 化の方向は正しいです。v6.7 では、同じガウスフェルミオン状態から幾何・エントロピー・物質・因果を読み出し、前線速度と Fermi 運動量の整合も確認しています。([GitHub][5]) v11.4 では、相互情報量から自然なテンソル分解を読み出す機構を導入し、RingChain、TfdPair、GrowingChain で状態ごとに異なる分解を発見する形に進んでいます。([GitHub][6])

第四に、陰性結果を理論の骨格にしている点が良いです。世代対 σ について、Wilson 線細分化、鏡映、共役、傾き T⁴ などの代替仮説を検定し、σ を「位置・向き・傾きに還元されない離散データ」と位置づけています。([GitHub][7]) これは物理研究として健全です。成功例だけを積むより、代替仮説を落とす方が理論の情報量は増えます。

ただし弱い点も明確です。

最大の弱点は、**ゲージ・物質・幾何・重力がまだ完全には同じ core から出ていない**ことです。v6.7 自身が、core はガウスフェルミオン系のみを扱い、φ⁴、格子ゲージ、アノマリー探索は枠外だと明記しています。([GitHub][5]) つまり「同じ網の異なる読み出し」という中心テーゼは、まだ全領域ではなく、ガウス系の成功例に留まっています。

次の弱点は、**SM の一意性は強いが、窓付き有限定理である**ことです。v14.5 も、扱う大表現は v6.2 の 10 型までで、さらに大きい型は窓の外だと明記しています。([GitHub][4]) したがって「指定窓内で SM が一意」と「自然界が SM を選ぶ原理」は別問題です。この差を埋めるのが今後の本丸です。

さらに小さいが重要な監査点として、v14.5 は Lean 定理が計 10 本と書いている一方、v15.0 は Lean 定理 9 本と書いています。([GitHub][4]) これは本質的な物理問題ではありませんが、主張台帳型の研究ではこういう不整合はすぐ直すべきです。外部査読者はこういう箇所から信頼性を判断します。

## 次に進めるべき研究プログラム

### 1. v16: 「監査版」を作る

まず新しい物理を増やす前に、**全主張の依存グラフ**を作るべきです。現在の claims.yml は C0〜C5 の分類として有用ですが、次は各 claim を DAG にしてください。

各 claim に対して、

`前提 → 計算 → 証明 → 結果 → 限界 → 反証条件 → 上位 claim`

を機械可読にします。

たとえば、

`QRN-GAUGE-***`
が
`表現窓の定義`
`アノマリー係数表`
`カイラル条件`
`全数探索完全性`
`同値軌道の定義`
`観測的に 16 成分隣人を排除する条件`

のどれに依存しているかを明示します。

目的は「正しさを主張する」ことではなく、**どの仮定を抜くとどの結論が落ちるか**を機械的に見えるようにすることです。これができると、QRN は単なる大きな研究ノートではなく、「反証可能な理論地図」になります。

成果物は次の形がよいです。

```text
claims.graph.json
assumptions.yml
falsifiers.yml
evidence_matrix.md
```

さらに、v14.5/v15.0 の Lean 定理数の不整合、README が v6.7 までに見える一方で docs は v15.0 まで進んでいる点など、外部から見た混乱を直してください。これは研究の中身以上に重要です。理論の第一印象は README で決まります。

### 2. v17: QrnCoreV2 を作る

いま最重要なのは、**ガウスフェルミオン core から、相互作用・ゲージ・制約付き Hilbert 空間を含む core へ拡張すること**です。

現在の v6.7 core は、

```text
QrnState = ガウスフェルミオン網
QrnModel = assumptions(), claims(), init(), evolve(t)
readout = entropy / MI / geometry / k_F
```

という形です。([GitHub][5])

次はこれを、

```rust
trait QrnState {
    type Site;
    type LocalHilbert;
    type Constraint;
    type Observable;
}

trait QrnDynamics<S: QrnState> {
    fn step(&self, state: &S, dt: f64) -> S;
    fn conserved(&self, state: &S) -> Vec<Invariant>;
}

trait QrnReadout<S: QrnState, R> {
    fn readout(&self, state: &S) -> R;
}
```

のように抽象化し、少なくとも次を同じ core に載せます。

```text
幾何 readout      : MI 距離、MDS、スペクトル次元
因果 readout      : Lieb-Robinson 前線、局所クエンチ速度
エントロピー      : 区間 entropy、相対 entropy、modular Hamiltonian
ゲージ readout    : Wilson loop、Gauss law、curvature
物質 readout      : chiral zero modes、世代数、anomaly
重力 readout      : QNEC gap、entanglement first law、Einstein residual
```

合格条件は、「別々の toy model を同じフォルダに置く」ではなく、**同一状態・同一発展・同一制約から複数 readout が出ること**です。v6.7 がガウス系でやったことを、ゲージ制約つき相互作用系へ広げる必要があります。

ここで重要なのは、最初から 3+1 次元の完全理論を狙わないことです。まずは 1+1D または 2+1D の小さい模型でよいです。ただし、次の陰性対照は必須にしてください。

```text
MI shuffle
GOE/random state
volume-law state
Gauss law を破った状態
wrong hypercharge
non-chiral mirror pair
wrong tensor factorization
```

v6.4 で MI シャッフルや GOE などの陰性対照を使って「検出しているのは MI 減衰の局所的パターン」と切り分けた姿勢は、そのまま core 全体に拡張すべきです。([GitHub][1])

### 3. v18: 「連続極限」を中心課題にする

統一場理論として認められるには、有限格子での PASS だけでは足りません。必要なのは、**格子サイズ、カットオフ、結合定数、粗視化に対して同じ物理が安定すること**です。

格子ゲージ理論では Wilson loop や閉じ込め、連続極限、有限サイズ scaling が中心的な診断になります。([arXiv][8]) QRN でも同じく、各 readout について

```text
N → ∞
a → 0
t/N 固定
mass gap → 0
critical point 近傍
coarse graining 反復
```

で何が保存されるかを見せる必要があります。

特に次を測るべきです。

```text
有効光速度の等方性
Lorentz violation residual
spectral dimension の flow
entanglement entropy の scaling
QNEC gap の N 依存
Wilson loop string tension の scaling
chiral zero mode degeneracy の安定性
anomaly cancellation の coarse-graining 不変性
```

目標は、「ある N で PASS」ではなく、

```text
residual(N) = A N^{-p} + B
```

を fit し、`B = 0` と言えるか、または `B ≠ 0` なら理論の残差として記録することです。

### 4. v19: σ・磁束・Wilson 線の選択原理を探す

v14.0/v15.0 が認めている通り、σ・磁束・Wilson 線・トーラス数を一括で決める上位原理はまだ残っています。([GitHub][9]) これは QRN の最重要課題です。

ここでやるべきことは、「もっと良い fit を探す」ではありません。やるべきことは、**選択原理の候補を有限個にして、全部落とすか残すかを決めること**です。

候補は例えば次です。

```text
最小記述長原理:
    データを説明する QRN 構造の Kolmogorov/MDL 的長さが最小

エンタングルメント安定性:
    σ・磁束・Wilson 線のうち、readout 間の整合 residual が最小

異常消去 + 世代縮退 + 湯川階層の同時最適化:
    anomaly, degeneracy, Yukawa likelihood を同じ objective に入れる

真空選択の熱力学:
    partition function / entropy / free energy で離散データを重み付け

計算可能性制約:
    無矛盾に符号化できる最小構造だけを許す
```

ただし、どれも C5 スローガンのままでは駄目です。各候補に対して、

```text
入力データ
自由パラメータ数
探索空間
null model
Bayes factor
外れたときの棄却条件
```

を定義します。

QRN は v6.5 で FN とアナーキーの Bayesian model comparison を行っているので、この方法を σ・磁束・Wilson 線選択へ拡張するのが自然です。README でも v6.5 は FN が Occam 罰込みでアナーキーに勝つ結果として整理されています。([GitHub][1])

### 5. v20: QNEC → 非線形 Einstein を攻める

QRN の「重力 = 情報単調性」路線で最大の未解決は、**線形化 Einstein から非線形 Einstein へ進めるか**です。v15.0 でも「QNEC → Einstein 全次数」は未解決として残っています。([GitHub][2])

既存文献では、Jacobson は地平線エントロピーと熱力学関係から Einstein 方程式を状態方程式として導く視点を与えました。([arXiv][10]) また、Faulkner らは holographic CFT において entanglement から重力方程式が出る関係を示しています。([arXiv][11]) QNEC については、エントロピーの null 方向二階変分とストレステンソルを結ぶ不等式として証明・研究されています。([arXiv][12])

QRN でやるべきことは、これらを引用するだけではなく、**非線形 residual を測る計算問題に変換すること**です。

例えば、

```text
EinsteinResidual[g, T] =
    || G_{\mu\nu}[g] + Λ g_{\mu\nu} - 8πG T_{\mu\nu} ||
```

を readout で定義し、

```text
entropy data → emergent metric g
local modular energy → T
QNEC / relative entropy monotonicity → inequality constraints
```

から residual が次数ごとに減るかを見る。

最初の目標は「非線形 Einstein を導出した」と書くことではなく、

```text
一次: PASS
二次: residual scaling が見えるか
三次: どの項が足りないか
```

を明確にすることです。

ここで陰性結果が出ても価値があります。むしろ「QNEC だけでは足りず、追加原理 X が必要」と分かれば、それは QRN 公理系を削るための情報になります。

### 6. v21: QRN 固有予言を作る

最終的に統一場理論として前進するには、**既存理論の再現ではなく、QRN だから出る数値予言**が必要です。v15.0 でも「重力以外の QRN 固有予言」は未解決のままです。([GitHub][2])

ここで重要なのは、予言を「当たるかもしれない言葉」にしないことです。次の形式に固定してください。

```text
observable:
    何を測るか

QRN value:
    QRN が出す値・範囲

competing models:
    SM, ΛCDM, FN, anarchy, GUT, string-inspired models など

data not used:
    fit に使っていない holdout data

falsifier:
    どの観測値なら QRN claim を降格するか
```

候補は例えば次です。

```text
ニュートリノ CP 位相 δ_CP の範囲
ニュートリノ質量階層
B−L に由来する sterile/right-handed sector の制約
CKM/PMNS の out-of-sample 予測
Yukawa texture のゼロまたは近似関係
rare anomaly-free neighbor の phenomenology
Lorentz violation residual の上限
重力誘起エンタングルメントの位相スケーリング
```

ただし、BMV 型の重力誘起エンタングルメント実験については慎重に扱うべきです。BMV は重力が entanglement を生成できるかを通して重力の非古典性を調べる有力な方向ですが、近年は「何をもって量子重力の証拠とみなすか」について理論的議論も続いています。([arXiv][13]) QRN はここを「実験の審判」として使うなら、単に entanglement 有無ではなく、**QRN 固有の位相、距離、質量、時間依存性**を出す必要があります。

## 研究の進め方として最も重要な規律

これ以降は、各バージョンを次の型に固定するとよいです。

```text
vX.Y:
    Question:
        何を解くか

    Claim to promote:
        C3 → C2 なのか
        C4 → C3 なのか
        C5 を削るのか

    Null models:
        何と比較するか

    Falsifier:
        何が出たら失敗か

    Artifacts:
        code
        result JSON
        proof
        certificate
        paper section

    Downgrade rule:
        失敗した claim をどの等級へ落とすか
```

特に、**C5 を増やさない**ことが重要です。QRN の現時点の価値は、C5 の大きな物語ではなく、C1/C2/C3 の executable atlas と、C4 の現象論を C2/C3 へ引き下ろそうとする姿勢にあります。

## 論文化の優先順位

v15.0 では、geometric-yukawa と anomaly-search の完全原稿が揃い、残るのは LaTeX 整形・投稿先決定とされています。([GitHub][2]) これはすぐ進めるべきです。

ただし投稿時の表現は、次のように切る方が通りやすいです。

**論文 1: anomaly-search**

主張は「指定された EFT 窓内で、アノマリー消去・カイラリティ・表現制約を満たす最小解が SM 1 世代として孤立している」です。これは最も強い一本です。v14.2 の構成案もこの方向です。([GitHub][14])

**論文 2: geometric-yukawa**

主張は「乱雑 O(1) 係数ではなく、幾何・Wilson 線・離散データから Yukawa 階層をどこまで説明できるか」です。v14.1 は、単一トーラス no-go、T²×T²、geometry selection、pairing as physical dof、honest-index alternative を含む構成になっています。([GitHub][15])

**論文 3 はまだ早いです。**

「Quantum Relational Network as a unified field theory」は、QrnCoreV2 がゲージ・物質・幾何・重力を同じ core で扱えるようになってからがよいです。今出すなら review/position paper にはなりますが、統一理論としては C5 が前に出すぎます。

## 逆に、やらない方がよいこと

今後避けるべきことは明確です。

新しい toy model を大量に足すことは、もう限界効用が低いです。README の初期ロードマップは「確立物理の再現 → 仮説要素の検証」を原則にしていましたが、v15.0 時点ではその段階をかなり越えています。([GitHub][16])

また、「SM が有限窓で一意」を「SM が導出された」と言い切るのも避けるべきです。正確には「この窓・この条件では一意」です。これを維持した方が、外部から見た信頼性は上がります。

さらに、「σ は既約な離散データ」と分かったあとに、別の自由データを足して fit を改善する方向も危険です。やるべきことは、自由データを増やすことではなく、**自由データを選ぶ原理を作ること**です。

## 最短ルート

最短ルートはこれです。

```text
v16: 全 claim の依存 DAG 化、README/claims/docs の監査、Lean 定理数など不整合修正
v17: QrnCoreV2 — 相互作用・ゲージ制約・物質 readout を同一 core に載せる
v18: continuum/RG scaling — N 依存と coarse-graining 不変性を測る
v19: σ・磁束・Wilson 線・トーラス数の選択原理を有限候補で競わせる
v20: QNEC/相対エントロピーから非線形 Einstein residual を次数別に測る
v21: QRN 固有の数値予言を pre-register し、holdout data で検定する
```

一言で言えば、**QRN は「世界を説明する物語」から「どの仮定を入れるとどの物理が出るかを数える機械」へ進めるべき**です。統一場理論に近づく道は、主張を大きくすることではなく、自由入力を減らし、同じ core から出る readout を増やし、失敗条件を鋭くすることです。
