# 4. “PASS/FAIL” を研究用に作り直す

現状の PASS/FAIL は魅力的ですが、査読には弱いです。

たとえば v4.1 の QNEC result では、基本 QNEC の最小ギャップは正ですが、強い QNEC では最小ギャップが -0.00003 でありながら PASS とされています。数値誤差として許容できる可能性はありますが、論文では tolerance、微分幅、有限サイズ効果、丸め誤差、N 依存性を出す必要があります。([GitHub][6])

QNEC 自体は既に量子場理論側で証明された既存結果なので、ここでの主張は「QNEC の発見」ではなく、

> 特定の自由フェルミオン toy model において、離散化・有限サイズ下で QNEC 型不等式を数値的に再現した。

にするべきです。QNEC は stress tensor と null deformation に沿った entropy の二階変分を結びつける条件として既存研究で定式化・証明されています。([arXiv][7])

追加すべき出力はこれです。

```text
PASS only if:
  min_gap > -tolerance
  tolerance = derivative_error + finite_size_error + roundoff_error
  convergence_rate is monotone or explained
  null control fails where theory predicts failure
```

各 simulation は、最低でも次を出してください。

```json
{
  "claim_id": "QRN-QNEC-001",
  "seed": 123456,
  "lattice_size": 302,
  "step_size": 0.01,
  "observable": "2*pi*Tkk - Spp",
  "min_gap": 0.00004,
  "estimated_error": 0.00001,
  "pass": true,
  "controls": [
    {"name": "non-chiral perturbation", "expected": "weaker agreement", "observed": "..."}
  ]
}
```

# 5. 標準模型探索は「証明に近い計算」にする

v3.1/v4.3 はこのリポジトリの中で最も論文化しやすい部分です。ここを本気で仕上げるべきです。

## 必要な改善

### 5.1 exact arithmetic にする

hypercharge、anomaly coefficients、Dynkin index はすべて有理数・整数で扱えるはずです。浮動小数を排除し、`i64` / `BigInt` / rational で処理する。外部依存なしを保ちたいなら自前 `Rational` を作ってもよいですが、論文用には検証用 backend として `num-rational` 版もあると安全です。

### 5.2 unsat certificate を出す

「解がない」は、単なる総当たりログよりも、証明 certificate がある方が強いです。

方法は 3 つあります。

1. 全探索結果の hash と canonical representative を保存する。
2. SMT / SAT / ILP に落として unsat certificate を保存する。
3. Lean / Coq / Isabelle で anomaly equation と探索空間の有限性だけを証明する。

完全形式化まで行かなくても、`certificates/` に以下を置くだけで価値が上がります。

```text
certificates/
  v31_domain.json
  v31_solutions.json
  v31_no_solution_below_15.sha256
  v43_domain.json
  v43_canonical_representatives.json
```

### 5.3 探索範囲を物理的に説明する

v3.1 は reps ≤ (3,2)、|Y|≤3/2、多重項 ≤5 という範囲で、v4.3 も reps ≤(3,2)、|Y|≤3/2、≤6 多重項などの制限を持ちます。ここを「計算都合」ではなく、「低エネルギーで小さい表現・小さい電荷が出やすい」という EFT / RG / phenomenology の仮定として明示する必要があります。([GitHub][4])

### 5.4 反例探索を自分でやる

査読者は必ずこう言います。

> 探索範囲を変えたら別解が出るのでは？

したがって、自分で以下をやる。

* SU(3) 表現: 1, 3, 3̄, 6, 6̄, 8
* SU(2) 表現: singlet, doublet, triplet
* hypercharge 範囲: |Y|≤2, |Y|≤3
* 多重項数: ≤7, ≤8
* vectorlike pair を許す場合と除く場合
* 右巻きニュートリノを含む場合
* 複数 U(1) を許す場合
* SU(5), SO(10), Pati–Salam から降りる場合

その上で、「SM はどの仮定に対して頑健で、どの仮定で崩れるか」を地図にする。
**崩れる条件を発見すること自体が価値**です。

# 6. Yukawa / FN 部分は Bayesian model comparison にする

v3.2 は、ε=0.22 と FN 電荷を入れ、9 個の質量比・CKM 量を実測の factor 5 以内で再現したとしています。これは面白いですが、現状では「選んだ電荷で合った」に見えます。results も ε の起源と電荷割当てが残る問いだと認めています。([GitHub][8])

ここは、次の形にすると価値が上がります。

## やるべきこと

* FN 電荷を全探索する。
* O(1) 係数の prior を明示する。
* 実測値との log-likelihood を定義する。
* null model と比較する。
* AIC / BIC / Bayes factor を出す。
* 学習に使った量と、予測に使った量を分離する。

たとえば、

```text
Model M0: all Yukawa entries are random O(1)
Model M1: FN with arbitrary integer charges, epsilon fitted
Model M2: FN with charges constrained by QRN geometry
Model M3: FN charges derived from same topological data as generations
```

とする。

現状の v3.2 は M1 です。
本当に QRN の価値にするなら M2 または M3 に進める必要があります。

# 7. 「単一 QRN dynamics」を作る

これは統一場理論として価値を上げるための本丸です。

いまは、多数の toy model が並んでいる状態です。
これを統一理論に近づけるには、**共通の状態空間と共通の更新規則**を定義する必要があります。

最低限、次の 4 層を分ける。

```text
QRN Core
  ├─ State: finite-dimensional Hilbert space / tensor factorization / operator algebra
  ├─ Dynamics: unitary update rule or variational principle
  ├─ Readout: geometry, gauge field, matter, entropy, causal order
  └─ Validation: known limits and falsification tests
```

たとえば、以下のような core trait を作る。

```rust
trait QRNModel {
    type State;
    type Observable;
    type Assumption;
    type Readout;

    fn assumptions(&self) -> Vec<Self::Assumption>;
    fn evolve(&self, state: &Self::State, steps: usize) -> Self::State;
    fn readout(&self, state: &Self::State) -> Self::Readout;
    fn claims(&self) -> Vec<ClaimId>;
}
```

そして、各 vXX binary は独立した小話ではなく、

```text
QRN core state
  → readout_geometry()
  → readout_gauge()
  → readout_matter()
  → readout_entropy()
```

のどれかとして再実装する。

統一理論としての説得力は、**新しい simulation を増やすことではなく、既存 simulation を同じ core から出すことで上がります**。

# 8. 反証条件を現代化する

v1.0 は BMV 実験を「重力が量子的でなければならない」反証条件として掲げています。BMV 型提案は、重力相互作用で 2 つの質量が entangle すれば重力の量子性を示せる、という方向で 2017 年に提案されました。([GitHub][9])

ただし、2025 年の Nature 論文は、古典重力でも full QFT 的な扱いでは quantum information を伝えて entanglement を生成し得る、と論じており、BMV の解釈は以前より微妙になっています。したがって QRN の反証条件は、単に「BMV entanglement あり/なし」ではなく、**どの locality assumption、mediator assumption、semiclassical gravity class を排除するのか**まで書くべきです。([Nature][10])

反証条件 table を作るとよいです。

| 観測                 | QRN が予測するもの                 | 競合理論でも可能か               | QRN が死ぬ条件                   |
| ------------------ | --------------------------- | ----------------------- | --------------------------- |
| BMV 型 entanglement | あり                          | 一部の古典/半古典 model でも可能性あり | QRN 固有の phase scaling が外れる  |
| 0νββ               | seesaw / Majorana と整合するなら期待 | 多くの BSM でも可能            | QRN が固定した mass ordering と矛盾 |
| proton decay       | GUT 的 embedding を採るなら期待     | SUSY/GUT 依存             | QRN の統一 scale と非整合          |
| w(z)               | 情報論的 Λ model があれば具体化        | ΛCDM でも説明可能             | QRN 固有の deviation が排除       |
| tensor B-mode      | inflation module と接続        | inflation models 多数     | QRN が固定する r 範囲が排除           |

「観測で白黒がつく」と言うには、**QRN 固有の数値予言**が必要です。

## 数値 library の検証

自前 RNG、複素数、Jacobi 固有値、Bessel 関数などを実装しているのは面白いですが、論文では「自前実装のバグではない」と示す必要があります。lib.rs には xorshift64* RNG、C64、三重対角 solver、Jacobi 固有値分解、行列関数、Bessel 関数などが含まれています。([GitHub][13])

対策は、

* 小さい行列は解析解と比較。
* 乱数は seed 固定。
* 別 backend と cross-check。
* numerical tolerance を明示。
* debug mode と release mode の差を検査。
* `f64` と high precision 版を比較。

# 11. 型で仮定を表現する

Rust を使うなら、物理的仮定を型に落とすと研究の質が上がります。

```rust
struct Hypercharge6(i32);      // Y * 6
struct ComponentCount(u32);
struct DynkinIndex(i64);

enum ClaimLevel {
    ExistingTheorem,
```

表現も型にする。

```rust
enum SU3Rep {
    Singlet,
    Fundamental,
    AntiFundamental,
    Sextet,
    AntiSextet,
    Octet,
}

enum SU2Rep {
    Singlet,
    Doublet,

    su3: SU3Rep,

```

これにより、`Y` の正規化ミス、共役の扱い、component count の取り違え、同値類の重複などを減らせます。

# 12. “負の結果” を入れる

研究として信頼されるには、成功例だけでは弱いです。

追加すべき negative controls は以下です。

## emergent geometry 系

* 相互情報量行列をランダム shuffle する。
* 同じ entropy spectrum だが局所構造を持たない状態を入れる。
* nonlocal Hamiltonian で進化させる。
* classical correlation だけの状態を入れる。
* volume-law random state を入れる。

> QRN readout は幾何を復元できない。

これが出ると、「都合よく何でも幾何に見えているだけではない」と言えます。

* anomaly 条件を 1 つ外す。
* chirality 条件を外す。
* 全因子荷電条件を外す。
* vectorlike pair を許す。
* hypercharge lattice を変える。

期待される結果：

> SM の孤立性がどの条件に依存しているかが見える。

## Yukawa 系

* FN charge をランダム化する。
* O(1) 係数だけで fit する。
* ε を自由にしすぎる。
* 学習量と予測量を入れ替える。

期待される結果：

> FN 構造が本当に情報を持っているか分かる。
