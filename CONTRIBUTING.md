# 貢献ガイド (CONTRIBUTING)

本リポジトリは物理学の研究プロジェクトであり、通常のソフトウェア開発とは規約が異なる。

## 原則

1. **外部クレート禁止**。必要な数値計算は `sim/src/lib.rs` に自作して追加し、
   `self_test()` に厳密値との照合を足す。再現性 (ネットワーク不要・供給網リスクなし)
   のための規約である。
2. **全シミュレーションに `[PASS]`/`[FAIL]` 判定を内蔵する** — 厳密解・観測値・
   双対性・別法・陰性対照のいずれかとの比較。乱数は固定シード。
   監査系バイナリは FAIL 時に非ゼロ終了する。
3. **主張台帳**: 新しい主張は `claims.yml` に等級 (C0–C5)・証拠・限界つきで追記し、
   `v61_ledger` が PASS であることを確認する。あわせて `claims.graph.yml` に
   依存 (deps)・仮定 (asm)・反証条件 (fal) を追記する — 辺は「それが落ちれば
   この主張も落ちる」関係のみ (動機・歴史の辺は張らない)。新しい仮定・反証条件は
   `assumptions.yml` / `falsifiers.yml` に登録し、`v151_audit --write` で生成物を
   更新して `v151_audit` が PASS であることを確認する (v15.1)。
4. **バージョンごとに 1 コミット**。メッセージは `vX.Y: 日本語タイトル`。
   README のバージョン表に行を追加する。
5. **知的誠実性**: 「教科書的事実」「確立された研究成果」「本プロジェクトの仮説」を
   区別する。失敗した検証・設計ミス・方法の限界も削除せず記録する。
6. **言語**: 文書・コードコメント・コミットメッセージは日本語。
7. **定理証明器**: シミュレーションは原則 Rust だが、定理証明器が得意な分野
   (探索空間の有限性・アノマリー方程式の形式化など) は Coq / Lean 4 等を
   導入してよい (CLAUDE.md 追加規則)。

## 陰性対照の文化

成功例だけの検証は弱い。新しい読み出し・fit を足すときは「壊れた入力で壊れること」
を示す対照を同時に足す (v6.4, v6.5 §3 参照)。
