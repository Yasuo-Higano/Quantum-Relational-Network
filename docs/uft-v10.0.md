## 2. 第十期の物語 — 発見は照合装置から来た

第十期の最大の出来事は計画されていなかった。論文図の作成 (v9.3 の準備) で湯川
パイプラインを python/numpy に独立再実装し、**図中の全数値を一次ソースと照合する**
検証を内蔵したところ、単一トーラスは厳密一致するのに積模型だけが 1.65 nats ずれた。
切り分けの果てに出てきたのは物理でもバグでもなく、**厳密に wrap 境界上に乗った
モード中心のソートが ±1e-16 の丸めで世代ラベルを巡回させる**という装置の性質だった
(v9.2)。

- 公表値は「ある特定の規約の値」として 10/10 厳密再現された — 乱れた計算ではない。
- 幾何選択の勝者と順位は両規約で不変 — **結論は頑健**。
- しかし絶対値は規約条件付き — claims 3 件に正誤表を追記した。

教訓は二つ:
1. **「固定シードで再現可能」は「規約に依らず再現可能」を意味しない。** 縮退帯の
   ソート順は下流で物理量になり得る (CLAUDE.md の落とし穴に追記)。
2. **独立実装は贅沢ではなく装置の一部である。** アノマリー探索は三重実装で守られて
   いたが湯川側は単独実装だった。図の照合という「軽い独立実装」で足りた —
   改良方針 (PROMPT/1) の設計思想がそのまま機能した実例。

方法論の対 (v9.1) も同型: v8.1 の「質量と混合の緊張」は、模型の性質ではなく
**測り方 (点評価) の人工物**だった。証拠は attainable set 全体を見る。
v6.5 (アナーキーの素朴 MC が証拠を過小評価) から数えて、「測り方が結論を作って
しまう」事例はこれで三度目 — 測定装置の監査は物理の一部である。
