## 1. 何をしたか

v6.3 の QNEC 誤差予算バイナリは、状態構築 (円環 GS + カイラル波束)・エントロピー・
光的エネルギー T₋₋・カイラル度を全て局所実装していた。本バージョンはこれらを
共有 core (`lib.rs`) に移した:

- **模型 `PacketRing`** (QrnModel 実装): RingChain の基底状態に、フェルミ面近傍の
  モード窓をガウス重みで束ねた波束対の回転を加えた励起状態。`standing` で
  非カイラル対照 (定在波) に切り替え。
- **読み出し 4 種** (QrnState のメソッド): `readout_bond_energy` (h = −2 Re C_{x,x+1})、
  `readout_current` (j = 2 Im C_{x−1,x+1})、`readout_null_energy` (T₋₋ 型、3 ボンド平均、
  T_kk=4T₋₋ 規格化)、`readout_chirality` (Σj/v_F Σh)。
- `RingChain::gs_bond_energy` (T₋₋ の基底状態参照値)。

## 3. core 化の付加価値 (v6.3 に無かった検査)

- 波束回転後の `purity_defect` = 1.0e-15 — 回転はガウス純粋状態を厳密に保つ
  (この検査が v6.3 の nocc バグを v6.7 で捕まえた; 今回は最初から通る)。
- 掃引最終時刻 (t=30) でも purity_defect = 8.5e-15 — 発展のユニタリー性。
- `readout_fermi_momentum` = π/2 (偏差 9e-16) — 波束は粒子数を保存する。

## 4. 意味 — core 移行地図の更新

core 模型は 4 つになった: **RingChain** (幾何読み出し) / **TfdPair** (ER=EPR) /
**GrowingChain** (宇宙論的成長) / **PacketRing** (QNEC・エネルギー条件)。
幾何・熱・成長・エネルギー条件という別々の顔が、同じ状態空間
(ガウスフェルミオン網 C_ij) と同じ読み出しの語彙で書けている —
「toy の集まり」から「一つの網の異なる模型」への移行 (改良方針 §7) がまた一歩進んだ。

残高 7 の残りは**動的テンソル分解**のみ。

## 5. 限界

- 回帰一致は N=302 のみ (v6.3 の N 掃引 202/402 と誤差予算の再導出は重複するので
  行わない — それらは v6.3 が引き続き担う)。
- PacketRing は 1 波束・固定窓幅 (v4.1/v6.3 の設定)。窓の一般化は必要になったとき。
