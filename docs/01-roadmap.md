## 進め方の原則

1. 各バージョンで「確立物理の再現」→「仮説要素の検証」の順に進み、**厳密解との比較(PASS/FAIL)**を必ず入れる。
2. バージョンごとに git コミットする。
3. 失敗や不一致も文書に正直に記録する(それ自体がデータである)。
