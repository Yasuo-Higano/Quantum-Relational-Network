## 1. 動機 — 次の方針 (PROMPT/2) の柱 1

改良方針 (PROMPT/1) の計算項目は v15.0 で完済した。次の方針 (PROMPT/2) は「シミュレーションを
増やす段階ではなく、統一理論として弱い箇所だけを狙って潰す段階」と定め、最初の的を
**監査版** とした: claims.yml は等級 (C0–C5) を機械検証してきたが (v6.1)、**主張同士の依存**は
文書の散文の中にしかなかった。「正しさを主張する」のではなく「どの仮定を抜くとどの結論が
落ちるかを機械的に見えるようにする」— これができて初めて、QRN は大きな研究ノートではなく
**反証可能な理論地図**になる。

## 2. 台帳の設計

### 2.1 辺の意味論 (最重要の設計判断 — ASM-EDGE-SEMANTICS)

`claims.graph.yml` の辺 `deps` は「**それが落ちればこの主張も落ちる**」関係のみ。
動機・歴史・引用の辺は張らない。例:

- v4.3 の全数探索 (GAUGE-006) は v2.3 の逆問題 (MATTER-002) に**動機づけられた**が、
  MATTER-002 が誤りでも v4.3 の走査は独立に成立する — **辺なし**。
- v10.3 の対の既約性 (YUK-009, C2) は対模型 (YUK-007, C4) の**文脈で**生まれたが、
  定理自体はゼロモード構成 (MATTER-001, C1) にしか依存しない — **辺は MATTER-001 へ**。

この規律が **等級単調性** を可能にする: rank(dep) ≤ rank(claim)、順位は
**C0 < C1 < C2 < {C3, C4} < C5** (機構 C3 と現象論 C4 は強さの順序を持たない横並び)。
つまり「SM 一意性の C2 定理が C4 の fit や C5 の解釈に依存しない」ことが**機械検証**される。

## 4. 依存グラフが見せたもの

**構造**: 節点 96 / 依存辺 111 / 根 (deps なし) 35 / 最大深さ 11 (例: C0-005 → … → META-013 の鎖)。
等級単調性の違反 0 — ただしこれは自明ではなかった。辺を「動機」で張ると
C2→C4 違反が 3 箇所生じ (YUK-009→YUK-007 等)、意味論を「落ちれば落ちる」に締めることで
初めて単調になった。**グラフを描く作業自体が、定理と物語の混線を 3 箇所ほどいた。**

## 5. 限界 (正直に)

- 辺の判定は人手であり、形式化されていない (ASM-EDGE-SEMANTICS)。監査が保証するのは
  **構造** (非循環・単調・被覆・参照実在) であって、辺の物理的正しさではない。
- 影響範囲 (閉包) は上界である。実際には部分的に生き残る主張がありうる
  (例: 解釈が落ちても窓付き計算定理は計算として残る)。
- `results/v150_full_suite.txt` の「9 本」誤記は書き換えずに残した — 結果ファイルは
  歴史記録であり、正誤は文書と新しい一次ソースが担う。
