## 2. キャッシュ・モードの独立検証 — 16/16 PASS

`verify_cache_modes.py`: sim/cache/ のモード表 (v16.11 導入、v16.11〜v16.13 の
一次計算が使用) を **numpy の複素エルミート定式化** (N²×N² の H_ij = −e^{−iθ}) で
検証した。Rust 側は 2N² の実対称埋め込み + 循環ヤコビ — 言語も定式化も違う経路である。
縮退部分空間の基底任意性を踏まえ、比較は基底非依存量のみ:

| 検査 | 結果 (4 標本: (s,k) = (0,0), (1,0), (3,7), (5,11)) |
|---|---|
| 縮退幅 (numpy) | ≤ 1.8e-15 (キャッシュ記録 ~5e-13 と整合) |
| ギャップ E₄−E₃ | 0.028983 — キャッシュ記録と 1e-9 で一致 |
| **射影子 ‖P_np − P_rust‖_F** | **~1e-11 (部分空間の一致 — 最強の検査)** |
| グラム行列 = I | ~1e-15 |

v9.2 の教訓「独立再実装は装置バグ検出器」の cache 版。**キャッシュが「加速器で
あって一次ソースでない」ことが、削除・再計算なしに直接検証された。**
