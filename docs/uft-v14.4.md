# 統一場理論 v14.4 — v4.3 域の Lean 化: 17 時間が 3.3 秒になり、毒が要らなくなった

**Version**: v14.4
**Date**: 2026-07-05
**成果物**: `proofs/AnomalyArray.lean` → `results/v144_lean.txt`
**対応残高**: 残高 8 (形式化 — 最大項目の解決)

---

## 1. 問題

v4.3 域 (小表現, |6Y|≤9, ≤6 多重項, ≤24 成分) の孤立性スペクトル {15:1, 16:8, 24:459}
は Rust 三重実装 + SHA-256 で確立済みだが、Lean 化は「燃料つき再帰 walk で ~10¹¹
呼び出し ≈ 17 時間」と見積もられ、残高に留まっていた。

## 2. 設計 — 配列ベースの中間一致 (MITM)

- **半分列挙**: 非減少な原子列 (長さ ≤3) を固定範囲 for ループの直積で構築 —
  **完全性が構造から自明**になり、燃料も毒値も不要になった (毒燃料設計より強い)。
- **結合**: 5 アノマリー和を単一 Int にパックした鍵で HashMap 結合
  (鍵の相殺・Witten パリティ・接合部の非減少・成分 ≤ 24)。
- 候補ごとに全条件を再検査し、正準形 (並べ替え・共役・U(1) 反転・**整数倍**) で
  重複除去してから成分数分布を数える。

**定理**: `v43_spectrum : arrayScan = (1, 8, 459, 0)` — 分布がちょうど
{15:1, 16:8, 24:459} で、それ以外の成分数の解は存在しない (第 4 成分 = 0)。

## 3. 結果

- **native_decide 3.3 秒** — 従来設計の予測 17 時間から**約 18,000 倍**の短縮。
- 核のみの decide で添字エンコードの検算 (SM が条件を満たす) も併置。
- Lean 定理は計 8 本になった (Anomaly 3 + AnomalyBig 2 + AnomalyArray 2 + SM 検算)。

## 4. 開発記録 — 定理の偽がバグを捕まえた

初回実行は `arrayScan = (1, 11, 594, 0)` で**定理が偽**: 正準形に gcd 約分
(U(1) 電荷の整数倍の同一視 — Rust 正準形は持っていた) が欠けており、倍電荷コピーを
別解として数えていた。「期待値を定理に固定してから走らせる」設計が、
照合を書く前にバグを検出した。修正 (gcdReduce) 後に成立。

## 5. 限界

- 信頼基盤は Lean 核 + native_decide (コンパイラ) — Anomaly.lean と同じ選択。
- 領域は v4.3 (小表現)。大表現込み ≤24 成分 (E1 域: {15:1,…,24:459} + 17/18/22) の
  配列化は同じ設計で可能になった (将来の拡張)。
