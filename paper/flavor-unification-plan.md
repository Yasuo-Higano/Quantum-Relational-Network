**登録日: 2026-07-23 (本ファイルのコミットがタイムスタンプ)。状態: 規則凍結・計算未実施。**
PROMPT/7 §6 第三優先の実行計画。対象: [geometric-yukawa-full.md](geometric-yukawa-full.md) /
[cp-complex-structure-full.md](cp-complex-structure-full.md) /

## 0. 動機 (PROMPT/7 の診断)

三部作は「独立した 3 つの確認」ではなく**一つのモデル開発過程**である: 矩形トーラスで
CP が出ないことを確認し (J=0 構造零)、complex structure を追加し、その後 measure と
補正クラスを検討した。各段階で**観測済みの失敗を使って次のモデルを選んでいる**ため、
各論文内の holdout だけでは研究プログラム全体の adaptive model-selection cost を
払っていない。統合論文は 3 本を「falsification-driven development」1 本に置き換え、
時系列の prequential 評価で全コストを支払う。

## 1. 評価式 (P2 — 凍結)

プログラム全体のスコアを

> S = Σ_k ln p(D_k | M_{k−1}, D_{<k})

で定義する。ここで:
- **D_k** = 段階 k で初めて評価に入った観測量ブロック (下の P1 で台帳から確定)。
- **M_{k−1}** = 段階 k−1 終了時点で**凍結されていた**モデル族 + 離散仮説空間 + 事前分布
  (該当コミットの claims/results から復元)。D_k を見た後の改変は M_k に属し、D_k の
  スコアには使えない。
- p は当時と同一の evidence 機構 (格子ゼロモード重なり + 離散データ周辺化 —
  v65/v81/v9x/v16x/v17x の装置) による事後予測密度。観測量の尤度幅も当時の登録値。
- **ベースライン**: 同じ D 系列に対する (i) anarchy (v14.1 論文の登録定義)、
  (ii) 自由電荷 FN (QRN-YUK-002 の定義)。報告量は S_model − S_baseline の総額と
  **段階別増分の全表** (符号を含め無条件公表)。

## 2. データブロックと凍結点の再構成 (P1 — 手続きの登録)

D_k と M_k の対応表は本計画では**手続きだけを登録**し、確定は台帳監査で行う
(記憶や論文の叙述ではなく git 履歴 + claims.yml + results/ を一次ソースとする):

1. 各観測量 (質量比 6・CKM 3・J・UT 角 2・その他) について「最初にモデル選択に
   影響した版」を claims/results の初出から特定する。
2. 各モデル要素 (矩形 / tilted / shear / 非対称 shear / τ_im / Wilson 格子 / pairing /
   測度候補 6 種) について「導入の根拠に使われた観測量」を docs の開発記録から特定する。
3. 上記 2 表から (段階 k, D_k, M_{k−1} の凍結コミット) を一意化し、
   `prequential_ledger.yml` として機械可読化する。曖昧な点 (例: ある観測量が
   「見られていたが選択に使われていない」ケース) は**保守側** (= 使われたとみなし
   D から除外、モデル側の情報として計上) に倒す。この保守規則も本項で凍結する。

暫定の段階像 (確定は上の手続き — ここは予想であり拘束しない):
M0 = FN/anarchy 基線 → M1 = 矩形 T²×T² (D1 = 質量比 6, D2 = CKM 振幅 3) →
M2 = shear complex structure (D3 = J; tilted の棄却込み) →
M3 = τ = 1/12 + i/2 (D4 = UT 角 holdout; 測度候補 6 種の棄却込み)。

## 3. 探索コストの会計 (P3 — 凍結)

各段階の離散仮説空間は「その段階までに**実際に試された**選択肢の集合」とし、
一様事前を課す (= 試行の Occam 代金)。集合の要素数は台帳から数える (選択ではなく
データ)。対象: 幾何族 (矩形/tilted/shear/非対称 shear/τ_im 走査点)・Wilson 線格子の
粗さ・世代 pairing の全列挙・測度候補 6 種・ソート規約等の「無垢に見えた仮定」で
後に仮説へ昇格したもの (geometric-yukawa §2.4 の系譜)。試行が results/ に痕跡を
持たない場合は docs の開発記録を数え、それも無い場合は数えない (過小計上側に倒す —
その旨を論文に明記)。

## 4. 成功規準 (P4 — 凍結)

- **主規準**: S_model − S_anarchy > 0 (プログラム全体が、支払った探索コストを含めて
  なお anarchy より速く学習した)。
- 段階別増分表は符号を問わず全公表。負の段階は「後続データで支払った」と論文の
  abstract レベルで明示する。
- 結果の如何にかかわらず統合論文は「三つの独立した成功」という枠組みを放棄する
  (成功規準を満たさない場合、その事実が論文の主結果になる)。
- 三部作の既存主張 (各論文内の holdout・+306 nats 等) は**段階内**の証拠として
  保持されるが、プログラム横断の証拠力は本監査の S のみが担う。

## 5. 覗き見なし宣言 (P5)

本計画のコミット時点で、上記 S・段階別増分・prequential_ledger のいかなる数値も
計算していない。実装 (次期版 v26x_prequential — 単一バイナリ、既存 evidence 機構の
再利用、決定的) は台帳凍結 (P1 の成果物コミット) の後に走らせ、初走を含む全走行を
results/ に保存する。ゲート較正で数値を見た後の規準変更は「開発記録」として残す
(v25.2-III run1 の前例に従う)。

## 6. 統合論文の骨子

- 題 (案): *Falsification-Driven Development of a Magnetized-Torus Flavor Model:
  A Prequential Audit of Adaptive Model Selection*
- 構成: 開発時系列そのものをデータとして提示 → prequential 形式論 → 段階別結果 →
  生き残ったもの (離散データ台帳 + τ = 1/12 + i/2 + THERMO の閾値下ヒント) →
  in-sample nats の証拠力の降格 → 三部作は補遺/コンパニオンに再位置づけ。
- 評価: 通常の in-sample fit ではなく prequential (P2)。「モデルがどれだけ良いか」
  ではなく「**プログラムがどれだけ正直に学習したか**」を主張の単位にする。

## 7. 完了条件と非拡張

本監査は既存結果の再評価であり、新しい物理モジュールを追加しない (PROMPT/7 §9 の
モラトリアム対象外)。完了 = prequential_ledger.yml + v26x_prequential + 統合論文
原稿 + claims (メタ主張 1 件, C4 級を上限とする)。三部作の個別投稿は本監査の
完了まで凍結する。
